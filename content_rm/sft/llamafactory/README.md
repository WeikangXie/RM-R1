# Ascend 910B LLaMA-Factory SFT

This directory contains the Ascend 910B training launcher for the business-domain content RM SFT task.

It is independent from the OpenRLHF launcher in `content_rm/sft/openrlhf/`. The shared contract is the reviewed SFT data generated by `content_rm/data/build_sft_dataset.py`.

## 1. Build LLaMA-Factory Data

From the RM-R1 repository root:

```bash
python3 content_rm/data/build_sft_dataset.py \
  --human-review content_rm/data/local/review/human_review.jsonl \
  --rubrics content_rm/data/rubrics.md \
  --llm-annotations content_rm/data/local/review/llm_annotations.jsonl \
  --output-dir content_rm/data/local/sft \
  --write-llamafactory-alpaca
```

This writes:

```text
content_rm/data/local/sft/llamafactory_alpaca/train.json
content_rm/data/local/sft/llamafactory_alpaca/test.json
content_rm/data/local/sft/llamafactory_alpaca/dataset_info.json
```

## 2. Train On Ascend 910B

On the company Ascend environment with LLaMA-Factory, CANN, and `torch_npu` available:

```bash
ASCEND_RT_VISIBLE_DEVICES=0 \
MODEL_PATH=/data/models/Qwen3-8B \
DATASET_DIR=/data/content_rm/sft/llamafactory_alpaca \
OUTPUT_DIR=/data/content_rm/checkpoints/qwen3-8b-lora-sft-ascend \
bash content_rm/sft/llamafactory/train_ascend_lora.sh
```

The launcher renders the YAML template into:

```text
${OUTPUT_DIR}/qwen3_8b_lora_sft_ascend.rendered.yaml
```

Then it runs:

```bash
llamafactory-cli train "${OUTPUT_DIR}/qwen3_8b_lora_sft_ascend.rendered.yaml"
```

## Useful Overrides

```bash
ASCEND_RT_VISIBLE_DEVICES=0,1
MODEL_PATH=/data/models/Qwen3-8B
DATASET_DIR=/data/content_rm/sft/llamafactory_alpaca
OUTPUT_DIR=/data/content_rm/checkpoints/qwen3-8b-lora-sft-ascend
CUTOFF_LEN=2048
PER_DEVICE_TRAIN_BATCH_SIZE=1
GRADIENT_ACCUMULATION_STEPS=8
LEARNING_RATE=1.0e-4
NUM_TRAIN_EPOCHS=3.0
LORA_RANK=8
SAVE_STEPS=100
EVAL_STEPS=50
```

For local config rendering without an Ascend runtime:

```bash
RENDER_ONLY=1 bash content_rm/sft/llamafactory/train_ascend_lora.sh
```

## Notes

- Default model: `Qwen/Qwen3-8B`; override `MODEL_PATH` on the company platform.
- Default mode: LoRA SFT, not full-parameter SFT.
- Default chat template: `qwen3_nothink`.
- The script tries to source `/usr/local/Ascend/ascend-toolkit/set_env.sh` when present.
- It checks `dataset_info.json`, `llamafactory-cli`, `torch_npu`, and `torch.npu.is_available()` before training.
